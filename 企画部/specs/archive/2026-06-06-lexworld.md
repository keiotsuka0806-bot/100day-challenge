# LexWorld 仕様書 v2

## 概要
ルールブロックがタイル上の物理オブジェクトとして存在するパズルゲーム。
プレイヤーはルールを押して並べ替え、論理の連鎖を組み立ててゴールを目指す。

## コアメカニクス（変わらない核心）

グリッド上に `IF [対象] → [結果]` 形式のブロックが置かれている。
プレイヤーはブロックを押して移動できる（Sokoban方式）。

ブロックが正しい順序で**隣接**すると連鎖が成立し、世界のルールが変わる。

```
[水→氷] 隣接 [氷→渡れる]  →  水タイルが渡れるようになる
[火→水] 隣接 [水→氷]      →  火タイルが氷になる
[空白→水] 単独             →  空白タイルが水タイルに変わる（世界を創る）
```

連鎖成立時：ブロック間を**電流アニメーション**が走り、タイルが変化する。
連鎖が切れた時：ブロックが暗く戻り、タイルも元に戻る。

## タイルの種類（4種に絞る）
| タイル | デフォルト |
|--------|-----------|
| 水 | 通過不可 |
| 火 | 死亡（その場でリスタート） |
| 岩 | 通過不可・押せる |
| 空白 | 通過可能 |
| ゴール 🚩 | クリア |

## ルールブロックの種類（6種）
| ブロック | 効果 |
|---------|------|
| 水 → 氷 | 水 = 渡れる（氷の見た目に変化） |
| 火 → 水 | 火 = 渡れる（水の見た目に変化） |
| 水 → 火 | 水タイルが火になる（罠） |
| 岩 → 消える | 岩タイルが消滅する |
| 空白 → 水 | 空白タイルが水になる（世界を創る） |
| 空白 → 火 | 空白タイルが火になる（世界を創る・罠） |

## ステージ設計

### 低（2連鎖・直線）
ルールブロック2個を正しい順で並べるだけ。邪魔なし。
例：川の前に `水→氷` と `氷→渡れる` が置かれている → 並べればゴール。

### 低（2連鎖・向き問題）
正しいブロックは揃っているが向きと位置の調整が必要。
余分なブロック1個あり。

### 中（3連鎖・世界を創る）
`空白→水` + `水→氷` + `氷→渡れる` の組み合わせ。
最初は何もないエリアを水→氷→道に変えてゴールへ。

### 中（3連鎖・罠あり）
`水→火` という罠ブロックが紛れている。
誤って繋ぐと安全だったタイルが危険になる。

### 高（4連鎖・逆転）
火タイルを `火→水→氷→渡れる` の4連鎖で渡る。
ブロックの置き順と押す順番の両方を考える必要がある。

### 高（4連鎖・世界改変）
空白エリアを水で満たし → 氷にして → 岩を消して → ゴールへ。
「破壊」と「創造」を組み合わせないと解けない。

## 技術スタック
- フロントエンド: Vanilla JS + CSS Grid + CSSアニメーション
- データ: LocalStorage（クリア状況）
- ホスティング: Vercel（静的デプロイ）
- 特殊ライブラリ: **なし**

## ファイル構成
```
開発部/lexworld/
  index.html    ← グリッドUI・ステージ選択画面
  app.js        ← ゲームロジック（移動・連鎖判定・Undo）
  levels.js     ← 6ステージ定義（配列）
  styles.css    ← グリッド・アニメーション
  manifest.json
  vercel.json
```

## 削除したもの（理由）
- Canvas API → CSS Gridで十分、実装量1/3
- Firebase → Vercel静的で十分
- sw.js → パズルゲームにオフライン不要
- ステージ10→6 → 丁寧に作った6個のほうが体験が良い
