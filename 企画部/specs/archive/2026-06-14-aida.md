# あいだ（Aida）

## 概要
大切な誰かと一緒にやる、相互理解の体験。二人が別々に・非公開で同じ深い問いに正直に答えると、AIが「本当は同じだった一点」と「言葉の奥の本心」を**双方向に翻訳**して両者に同時に返す。普通の会話では一生たどり着けない理解に、5分で届く。AIが人の代わりに話すのではなく、**人と人の"あいだ"に立って橋になる**プロダクト。

## コアバリュー（妥協しない一点）
- 一人 × AI × 1回出力の檻を出る。AIは**二人のあいだに立つ通訳**。
- AIは要約も判定もしない。返すのは ①隠れた共通点 ②各自の言葉の奥にある本心の翻訳 ③押し付けない次の一歩。
- 同時開示で「そんな風に思ってたんだ」を起こす。これが体験の心臓。

## ユーザーフロー
1. 関係を選ぶ（恋人 / 親子 / 友人 / 距離ができた人 / 仕事仲間…）→ 問いのセットが変わる
2. Aが部屋を作る → 4〜6個の問いに**非公開で**回答 → 共有コード／リンク発行
3. Bがコードで参加 → 同じ問いに**非公開で**回答（相手の回答は見えない）
4. 両者完了 → AIが翻訳を生成 → 二人に「あいだ」を**同時に開示**
5. 保存（任意）／別の人ともう一度

## 問いの設計（例・関係種別で言い回しを変える）
- 最近、相手に言えていないことは？
- 相手にいちばん分かってほしいことは？
- 相手のどんなところに、実は救われている？
- この関係がどうなったら、あなたは嬉しい？
- 言い過ぎた／言えなくて後悔していることは？

## AI出力の設計（心臓部）
**入力**: 二人の非公開回答（A・B）
**出力（両者に同じものを提示）**:
1. ふたりの「あいだ」にあった**共通点**（本当は同じことを思っていた一点）を1〜2個、具体的に
2. **Aの言葉 → 本心の翻訳** / **Bの言葉 → 本心の翻訳**（各自の防御的な言い回しの奥にある気持ちを、やさしく）
3. 次に二人でできる**小さな一歩**を1つだけ（命令でなく提案）

**ガードレール（絶対）**:
- 感情を**捏造しない**。書かれていないことを足さない（全社・嘘禁止ルールに直結）
- どちらの**肩も持たない**。優劣・正誤の判定をしない
- 診断・断定をしない。相手への否定的評価を**増幅しない**
- 危険（DV・自傷の示唆等）を検知したら、翻訳より先に専門の相談窓口を案内する

## 技術スタック
- フロントエンド: Vanilla JS（組織標準・ビルドなし）
- 部屋・回答の共有: **Firestore**（クライアントSDK。ルームdocに両者の回答と結果を保持）
- AI翻訳: **Vercel Serverless Function `/api/translate`**（`ANTHROPIC_API_KEY` はサーバ側に隠す。BYOK・キー直書きは禁止）
- ホスティング: Vercel
- PWA: manifest.json + service worker

## プライバシー / 倫理
- 回答はその部屋の二人だけのもの。第三者・学習に出さない
- センシティブな内容なので保持は最小限（結果表示後の保存は任意、一定期間で破棄を検討）
- 「感情を捏造しない」を品質の最優先KPIにする

## 難度
**高（1日フル）**

## 独自性（使う理由）
「ChatGPTに二人の会話を貼って仲裁させる」に似ているが、それは"片方が両方の本音を晒す"前提で成立しない。あいだは**お互い非公開で出し、AIが双方向に翻訳して同時に開示する**から、身構えずに本音が出る。Keiが、距離のできた誰か（家族・友人・かつての大切な人）と一度ちゃんと分かり合いたい——その今に効く。

## 20年後の視点
2045年、AIは人の代わりに話す存在になりがち。だが「人と人をもう一度つなぐ」用途こそ、AIが人間性を支える最良の形として残る。

## 備考
全組織の絶対ルール（出せる全力 / 面白さに本気 / 妥協しない / 嘘をつかない）に沿って起案。AI出力の誠実性が品質の核。Keiの最深テーマ「深く理解されたい」と「AIと人間の共生」に直結する旗艦案。
