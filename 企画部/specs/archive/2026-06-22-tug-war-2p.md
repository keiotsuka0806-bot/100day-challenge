# 綱引きスマホ（TugWar2P）

## 概要
2台のスマホをQRでつなぎ、端末を“振る・傾ける”力で綱を引き合うリアルタイム対戦。画面の中ではなく、2人の腕の動きそのものが勝負になる物理対戦ゲーム。

## MVP機能
- ホストがQRで部屋を作り、相手がスキャンで参加（Firestoreで同期）
- DeviceMotion（加速度）を引っ張り力に換算→中央のロープ位置をリアルタイム同期
- 3本先取＋勝敗演出。タイブレークで「猛ラッシュ」ゾーン

## 技術スタック
- フロントエンド: Vanilla JS / DeviceMotion API
- データ: Firestore（共有倉庫 tatsuro-kei-2026 に tugwar コレクション追加）
- 特殊ライブラリ: qrcode-generator

## 難度
**中〜高（半日〜1日）**

## 独自性
オンライン対戦は画面内の操作が普通。これは“自分の腕を振る運動量”が勝敗を決めるので、その場の2人が本当に盛り上がる体験になる。リンクを渡すだけで始まる手軽さ。

## 20年後の視点
2045年、体の動きで遊ぶのは標準。だが「ただのスマホ2台が綱になる」発想は、専用機材ゼロで身体性を取り戻した素朴な実験として面白がられる。

## 備考
iOSはDeviceMotionに許可ボタンが要る（ユーザー操作起点で `requestPermission`）。Face Magic Battleの2台対戦と同期基盤を共用できる＝技術の再利用が効く。
