# UptimeMini 仕様書

## 概要
監視したいURLを登録するだけでブラウザから死活監視し、シンプルなステータスページを自動生成。チームや友達に「今サービスが生きてるか」を共有できる軽量ステータスページツール。

## リサーチ根拠
- "Status page tools" は2026年の実績あるmicro SaaS パターン（bigideasdb.comで言及）
- Healthchecks.ioが$5K-$30K MRRをソロ開発で実現した先例
- 個人開発者・小規模チームにはStatusPage.ioのような大型SaaSは不要 → 「URLを貼るだけ」の簡易版に需要

## ターゲットユーザー
個人開発・小規模チームで「サービスが生きてるか手軽に共有したい」開発者。

## MVP機能一覧
- [ ] URL登録（最大5件）
- [ ] ブラウザから定期fetch（Service Worker経由、5分間隔）でHTTPステータス確認
- [ ] ステータスページ自動生成（グリーン/レッドバッジ + 最終確認時刻）
- [ ] 過去24時間のアップタイム率表示
- [ ] ステータスページURLをシェア（URLパラメータに設定をエンコード）
- [ ] ダウン検知時にブラウザ通知
- [ ] PWA対応

## 技術スタック
- フロントエンド: Vanilla JS + Service Worker
- データ: localStorage（監視設定・履歴）
- ホスティング: Firebase Hosting

## 画面構成
- ダッシュボード: URLリスト + ステータスバッジ + アップタイム率
- 公開ステータスページ: URLパラメータから設定を読み込む軽量ビュー

## デプロイ先
- Firebase project: 新規作成
- URL: https://uptime-mini-[id].web.app
