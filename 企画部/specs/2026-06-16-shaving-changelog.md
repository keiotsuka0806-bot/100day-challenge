# けずりログ（ShavingChangelog）

## 概要
GitHubの大量の autosave コミットを貼ると、「今日その人が実際に何を変えたか」を人が読める3行に圧縮する。

## MVP機能
- コミットログ（またはdiff要約）を貼る
- "意味のある変化"だけ抽出して3〜5行に
- 日報・note素材としてコピー

## 技術スタック
- フロントエンド: Vanilla JS
- データ: なし
- 特殊ライブラリ: OpenAI API

## 難度
**中（半日）**

## 独自性
「GitHubのログは autosave 祭りで読めない。これは"人に語れる今日の変化"だけを残す」

## 20年後の視点
2045年、作業の大半はエージェントが刻む。人間が"物語として振り返れる粒度"へ翻訳する層が要る。

## 備考
乗り換え理由: Keiのログは autosave だらけで日報に書きにくい。ただし日報運用は既に仕組み化済みで、アプリより /retrospective スキル拡張で足りる可能性がある。
