# 衝動ブレーキ（ImpulseBrake）

## 概要
「買う直前」に**商品ページのスクショを1枚貼る**（＝その買い物の「顔」になる）と、24時間後の自分になりきったAIが3つだけ問いを投げ、**あなた自身の"後悔率"**を突きつけ、24時間クールダウンをかける。商品名の手入力は不要、値段は任意。全ショッピングサイト共通で使え、iPhoneのショートカットでAmazon・楽天等のアプリ起動時に自動で立ち上がる。

## ターゲットユーザー
ネット通販でつい衝動買いして後悔しがちな人（特にiPhoneユーザー）。

## MVP機能一覧
- [x] 貼る: **スクショ（任意）＋一言メモ（任意）＋値段（任意）**。スクショはサムネイル化して端末内保存＝その買い物の「顔」。商品名の手入力は不要
- [x] 後悔率の提示: 過去の「勢いで買って後悔した割合」を決断の瞬間に表示
- [x] AI3問: 24時間後の自分視点で問う（OpenAI、鍵なしはモック。メモを手がかりに生成）
- [x] 24hクールダウン: 「冷やす」で"冷やし中"リストへ。タイマー後に見送る/やっぱり買う
- [x] 見送り集計: 値段を打てば「見送った金額¥」、打たなければ「見送った回数」を集計（達成感に変える）
- [x] 後悔チェックイン: 「買う」を選んだ商品に数日後、スクショ付きで「後悔した？よかった？」と聞き後悔率を育てる
- [x] 初期シード: 初回に直近の衝動買いを数件振り返って後悔率の初期値を作る
- [x] ショートカット設定レシピ: Amazon・楽天ほかを開くと自動起動する手順書を同梱
- 注: スクショからの商品名・値段の自動読み取り(AI Vision)は将来拡張。MVPはVision不要＝鍵なしでも全機能成立（コストゼロ公開可）

## 将来拡張（MVP後）
- Chrome拡張（PCブラウザのAmazon購入ボタン割り込み）
- Androidネイティブ（使用状況アクセス＋オーバーレイで自動起動）
- iOS Screen Time(Family Controls)による本格的な一時ブロック

## 技術スタック
- フロントエンド: Vanilla JS / PWA（manifest + service worker）
- データ: localStorage（端末内完結・外部送信なし）
- AI: Vercel Function経由でOpenAI（gpt-4o-mini）。鍵あり=本物/なし=モック
- ホスティング: Vercel
- 自動起動: iOSショートカット オートメーション（ユーザーが1回設定）

## データモデル（localStorage: impulse_brake_v1）
- items: [{ id, name, price, memo, status: 'cooling'|'bought'|'skipped', createdAt, cooldownUntil, regret: null|true|false, ratedAt }]
- 派生: 後悔率 = 評価済み'bought'のうちregret=trueの割合 / 見送り累計 = 'skipped'のprice合計

## 画面構成
1. ホーム（後悔率・見送り累計・冷やし中リスト・「買い物を止める」ボタン）
2. ポーズ（貼る→後悔率提示→AI3問→冷やす/それでも買う）
3. 後悔チェックイン（数日経過したboughtに後悔を尋ねる）
4. 初期シード（初回オンボーディング）
5. ショートカット設定手順

## 独自性
「家計簿が買った"後"を記録するのに対し、買う"前"の一拍だけに全振り」。さらに"自分の後悔率で自分を止める"点が新しい。thought-second-opinionと同じ心の支え柱。

## 20年後の視点
消費が即時最適化される世界で「あえて一拍おく」道具は、人が自分の意志を取り戻す小さな抵抗として残る。

## 備考
研究起点: DeepMind系 test-time compute（考える時間で判断の質が上がる）を人間の衝動買いに翻訳。全ショップ対応は"貼るだけ"設計で担保、自動起動はショートカットに委譲することで本体をVanilla JSに保つ。
