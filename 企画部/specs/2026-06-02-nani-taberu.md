# 今日なに食べる？ 仕様書

## 概要
毎日の献立を一汁三菜単位で提案するアプリ。Rakuten Recipe APIから幅広いレシピを取得し、PFCバランスと過去の食事履歴を元に栄養が偏らず・被らない献立をレコメンドする。

## ターゲットユーザー
「毎日の献立を考えるのが面倒」「なんとなく栄養が偏っている気がする」と感じている人。

## MVP機能一覧

### 献立提案（一汁三菜）
- [ ] 「決める！」ボタンで一汁三菜をまとめてランダム提案（スロット風アニメーション）
- [ ] 提案セット構成：ごはん（固定）+ 汁物1 + 主菜1 + 副菜2
- [ ] セット全体のPFC合計を表示
- [ ] 各料理に「レシピを見る」ボタン → 材料一覧・手順をモーダル表示（Rakuten APIのデータ）
- [ ] 「汁物だけ変える」「主菜だけ変える」などパーツ単位の再抽選ボタン
- [ ] 「この献立で食べた！」ボタンで一括記録

### 気分バイアス（重みづけ）
- [ ] 「今日の気分」をタグ形式で複数選択（任意）
  - **料理系**: 和食・洋食・中華・イタリアン
  - **食材系**: 肉・魚・野菜・卵・豆腐
  - **状態系**: さっぱり・がっつり・温かい・冷たい
- [ ] 選んだタグに合う主菜の抽選確率を3倍に重みづけ（除外はしない）
- [ ] 調理時間フィルター（15分以内 / 30分以内 / 制限なし）

### 栄養バランス考慮
- [ ] 直近3日間のPFC合計から不足栄養素を優先した主菜を重みづけ
- [ ] 本日の朝・昼・夜で記録し、1日のPFC達成率をリアルタイム表示
- [ ] 1日のPFC目標（デフォルト: P 120g / F 50g / C 250g）はユーザーが変更可能

### 履歴・被り防止
- [ ] 食べた献立をFirestoreに記録（日付・食事タイミング・各料理名・PFC値）
- [ ] 直近7日間に食べた主菜は提案確率を下げる
- [ ] 過去の食事カレンダーを表示（日付タップで献立詳細を確認）

### レシピデータ（Rakuten Recipe API）
- [ ] アプリ起動時にカテゴリ別レシピをAPIから取得・localStorageにキャッシュ（TTL: 24時間）
- [ ] 取得カテゴリ:
  - 汁物: 味噌汁・スープ・豚汁など
  - 主菜: 肉料理・魚料理・卵料理・豆腐料理
  - 副菜: 野菜炒め・和え物・煮物・サラダなど
- [ ] APIから取得する情報: レシピ名・材料・手順・画像URL・調理時間
- [ ] PFCは主要食材ごとの概算テーブル（`pfc-table.js`）を使って材料リストから自動計算

### 認証
- [ ] Firebase Auth（Googleログイン）でデータをユーザーごとに管理

## 将来拡張（MVP後）
- カスタム献立の保存（「この組み合わせが好き」）
- 週単位の栄養レポート・グラフ
- 家族・パートナーと献立を共有
- 外食・テイクアウト対応
- 買い物リスト自動生成（献立の材料をまとめる）

## 技術スタック
- フロントエンド: Vanilla JS + CDN
- レシピAPI: Rakuten Recipe API（無料・要アプリID登録）
- データ: Firebase Firestore（食事履歴・ユーザー設定）
- 認証: Firebase Auth（Google Sign-In）
- ホスティング: Firebase Hosting
- キャッシュ: localStorage（APIレスポンスを24時間キャッシュ）

## 事前準備（開発前に必要）
1. [楽天ウェブサービス](https://webservice.rakuten.co.jp/) でアプリIDを取得（無料）
2. Firebase プロジェクトを新規作成

## ファイル構成
| ファイル | 役割 |
|---------|------|
| `index.html` | 全UI |
| `app.js` | アプリロジック・状態管理・イベント |
| `rakuten-api.js` | Rakuten Recipe API取得・キャッシュ管理 |
| `pfc-table.js` | 食材別PFC概算テーブル（材料名→PFC計算） |
| `recommender.js` | 気分バイアス・履歴重みづけ・抽選ロジック |
| `styles.css` | 全CSS |

## 画面構成

1. **ログイン画面** — Googleログインボタン
2. **ホーム画面（メイン）**
   - 今日のPFC達成状況バー（朝・昼・夜の合計 vs 目標）
   - 気分タグ選択 + 調理時間フィルター
   - 「決める！」ボタン → スロットアニメ → 一汁三菜セット表示
   - 各料理カード（料理名・PFC・「レシピを見る」・パーツ再抽選）
   - 「この献立で食べた！」ボタン
   - レシピモーダル（材料・手順・画像）
3. **履歴画面** — カレンダーで過去の献立を確認
