# 使いどき診断（TsukaiDoki）

## 概要
作った企画やツールの説明を貼ると、「あなたが既存の定番ではなくこれを使う場面を1つ書けるか」を診断し、書けなければ弱点を指摘する。

## MVP機能
- 企画概要を貼る
- 類似の定番サービスを推定して提示
- "乗り換え理由"が言語化できているか採点＋改善質問

## 技術スタック
- フロントエンド: Vanilla JS
- データ: なし
- 特殊ライブラリ: OpenAI API

## 難度
**中（半日）**

## 独自性
「アイデア検証ツールは"良し悪し"を採点するが、これは"既存の定番を捨ててまで使う理由があるか"の一点だけを突く」

## 20年後の視点
2045年、誰でも一瞬でアプリを作れる。だから"作る価値の有無"を最初に弾く関所の価値は上がる。
