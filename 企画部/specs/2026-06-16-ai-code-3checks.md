# AIコードの危険3点チェック（AICode3Checks）

## 何ができる（1行）
AIが書いたコードの差分を貼ると、「ここだけは人間が必ず確認すべき危険3点」を、非エンジニアにも分かる日本語で教えてくれる。

## 概要
AIが大量にコードを書く時代に「全部読む」のは無理。削除・課金・認証・外部送信・データ書き換えなど“事故るとヤバい変更”だけを抜き出し、何を確認すべきかを平易に提示する。

## MVP機能
- diff/変更内容を貼り付け
- リスクの高い変更を自動抽出（削除/課金/認証/外部送信/DB書換）
- 「確認すべき3点」を平易な日本語＋確認方法で提示

## 技術スタック
- フロントエンド: Vanilla JS（or Next.js）
- データ: なし
- 特殊ライブラリ: OpenAI API

## 難度
**中（半日）**

## 独自性
「AIコードレビューは“全部詳しく”指摘して非エンジニアには重い。これは“危ない3点だけ見ろ”と絞り、コードが読めなくても判断できる」

## 20年後の視点
2045年、コードはほぼAIが書く。人に残るのは“どこを信じ・どこを疑うか”で、その補助線の価値は増す。
