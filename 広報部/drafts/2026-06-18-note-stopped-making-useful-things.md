# タイトル案（3つ）
1. 「便利なもの」を作るのをやめた日──AI会社の最終候補が3つとも全滅した話
2. 机上で選んだ“勝ち筋”が、実機で触った瞬間に全部ハズレた
3. 役に立つアプリより、心が動くアプリ。1日で方針が変わった理由

## リード（無料部分の冒頭）
非エンジニアの私が、Claude Code を「社員」にしてAIだけの開発会社を回しています。毎朝、企画部が10個のアイデアを出し、3つに絞り、その日のうちに1本作って公開する。昨日もそのはずでした。ところが、机上で慎重に選んだ最終3案を実際にスマホで触った瞬間、私の口から出たのは「……で、これ何がいいんだっけ?」。3つとも、でした。今日はその“全滅”から、会社の作るものの基準そのものが変わった話を書きます。

## 本文構成

### 1. 朝会という儀式
- 毎朝AIの各部署が動く流れを1段落で（企画→絞り込み→開発→QA→公開）
- 昨日の最終3案は「OGP画像メーカー」「アイコン一括生成」「作品集ジェネレータ」。どれも“便利な道具”

### 2. 実機で触った瞬間に崩れた
- 画面の中では正しく動く。なのに自分が使う場面が思い浮かばない
- 気づいたこと: アイコンを量産する「面倒」は、私の痛みではなく作る側（AI）の作業だった。私は一度もその不便を感じていない
- [Keiの言葉: ここに「で、何がいいの?」と言った時の正直な気持ちを1〜3文。がっかり/焦り/それとも妙な納得だったか]

### 3. 「乗り換える理由」というたった一つの問い
- 既存の定番でなく“これ”を使う理由を1文で書けないものは作らない、という社内ルールの紹介
- 便利さの差分では、人は乗り換えない。ましてや心は動かない

### 4. 道具をやめて、体験に振った
- その場で方針転換。作ったのは「ありがとうの貯金箱」──感謝をひとつ入れると光が落ちて溜まり、自分だけの小さな宇宙が育つ
- 役に立つかと聞かれると困る。でも、見た人がスクショして誰かに見せたくなる
- その日のうちに公開まで到達（鍵もサーバー代もかからない作りで）

### 5. 会社の基準が変わった
- 「便利か」ではなく「見知らぬ誰かの心が動くか、シェアされるか」を最上位の評価軸にルール改定
- AIは10案でも100案でも出せる。だからこそ“何を選ぶか”という人間の判断が会社の個性になる
- [Keiの言葉: 「便利なものより心が動くもの」を選びたいと思う、あなた自身の理由・原体験を2〜4文。ここが記事の核なので必ず自分の言葉で]

## 想定読者と狙い
- N=1ペルソナ: AIで何か作ってみたいが「結局よくある便利ツールの劣化版になる」ことに薄々飽きている個人開発者・非エンジニア
- 無料公開（コンテスト #AIで遊ぼう を意識）。マネタイズはこの記事単体でなく、「AI会社経営の実況」シリーズへの導線として
- 狙い: 「AIに作らせる時代の差別化は“判断軸”にある」という視点を、失敗談で腹落ちさせる。説教でなく、自分も全滅した当事者として書くことで刺さる
