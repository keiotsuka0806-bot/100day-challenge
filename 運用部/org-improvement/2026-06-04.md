### Taiichi Ohno の視点：パイプラインに在庫を積むな、流れを作れ

**思考原理**: トヨタ生産方式の本質は「必要なものを、必要なときに、必要なだけ」。在庫（=未使用の仕掛かり）はムダである。プッシュ型（毎日アイデアを生成して溜める）よりプル型（必要な時に必要な形で取り出せる）の方が効率が高い。

**現状の問題**: 企画部はアイデアを毎朝プッシュ生成する設計だが、ideas/フォルダはゼロ稼働。しかし本当の問題はそこではない——1人スタジオで1日1プロジェクトなら、アイデアの「在庫」より「プロジェクト開始の摩擦ゼロ化」の方が価値が高い。白紙からアイデアを選ぶより「このタイプのアプリなら30秒でCLAUDE.md+フォルダ構成が決まる」テンプレートがあれば、朝の最初の1時間を実装に使える。

**推奨アクション**: `企画部/templates/` に「プロジェクトアーキタイプ別テンプレート」を3〜4個作成する。毎日のアイデア生成より確実に価値が高い。

**具体的な実装**:

```
企画部/templates/
  vanilla-firebase-pwa.md   ← WhiskyNote型（vanilla JS + Firestore + Auth）
  react-local-pwa.md        ← MoodForecast型（React+Vite + localStorage）
  vanilla-static.md         ← Firebase不要の静的サイト型
```

各ファイルの中身（例: `vanilla-firebase-pwa.md`）:
