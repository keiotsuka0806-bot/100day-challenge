# Genius Council — 段取り（Dandori / 引越し手続きナビ） 2026-06-23

種別: 業務ツール／管理アプリ
参加プロ集団: SaaS PM・業務改善コンサル・アクセシビリティ専門家・QAエンジニア

## Loop 1: First-Use / Core Value
- 所見: 「リストを作る」直後に長い一覧が出て圧倒される懸念。
- 改善: リスト冒頭に「あなたの回答に合わせて N 件を用意しました。上から順にチェックすれば漏れません」の案内を追加し、初見の不安を解消。

## Loop 2: Expert Quality / Domain Depth
- 所見1: 社会保険（会社員）の人に健康保険まわりのタスクが一切出ず、抜け落ちる。
  - 改善: 「勤務先へ住所変更を届け出る（健康保険・住民税・通勤手当は会社経由）」を追加（insurance=shahoで表示）。
- 所見2: 各種住所変更で何度も使う「住民票の写し」を複数取得する助言が抜けていた。
  - 改善: 「住民票の写しを数通もらっておく」を当日〜14日以内に追加（常時表示）。
- 正確性方針: 期限・窓口・持ち物は人手でキュレーションし、AIに事実生成させない。自治体差があるため全画面で「最終確認は各自治体・公式で」を明記。

## Loop 3: Replay / Share / Operation
- 進捗をlocalStorageで保持（リロード・再訪で復元を実機確認）。
- 「完了を隠す」で残タスクだけに集約、全完了時は祝福の空状態。
- 共有（Web Share／クリップボードのコピー）・印刷用CSSを用意し、家族との共有や紙運用に対応。
- アクセシビリティ: 進捗ラベルに aria-live、チェックボックスに aria-label、選択肢はlabelで操作可能。

## 検証
- フロー（イントロ→質問→リスト→チェック→進捗）をPlaywrightで通し確認。
- スマホ幅320pxで横はみ出しなし・レイアウト保持を確認。
- コンソールエラーは favicon 404 のみ（無害）。

## QA / セキュリティ（デプロイ前最終）
- 外部送信なし・APIキーなし・バックエンドなし＝コストゼロ（Anthropicクレジット不使用）。
- innerHTMLは静的データのみ＋esc()でエスケープ。ユーザー入力（ラジオ）は固定値でHTMLに展開しない＝XSS面なし。
- 外部リンクは target=_blank rel="noopener noreferrer"。SWは同一オリジンGETのみキャッシュ（network-first）。
- 判定: 重大な指摘なし。共有可能品質。

## 次の増築
- 出産／結婚／身内が亡くなったとき を同じ枠組み（質問→条件分岐→時期別リスト）で追加。
