# TATSURO LP Workflow

## このLPで固まった設計ルール

- 商品写真を主役にする。説明は写真の意味を補う役に回す。
- ページ全体の重心は中央に置く。SaaSのように左寄せで情報を積む構成は避ける。
- 見出しに句読点は入れない。強いコピーは短く切る。
- 本文の改行は人間が決める。単語の途中、文の途中では切らない。
- 改行は句読点の後に寄せる。意味のまとまりを壊さない。
- セクションごとに役割を分ける。Hero, Land, Craft, Taste, Philosophy, CTA の順で深める。
- 同じ画像を使い回さない。セクションごとに意味の違う写真を置く。
- 余白は広すぎても狭すぎても破綻する。見出し・本文・画像・CTAの箱幅を揃える。
- CTAは横に広げず、中央列に縦積みで置く。
- 酒類LPは年齢確認と注意書きを必ず入れる。

## 天才集団

| 役割 | 見ること | 成果物 |
| --- | --- | --- |
| ブランド戦略家 | 誰に何を約束するLPか | ターゲット、ポジショニング、購入理由 |
| コピーライター | 一瞬で残る言葉か | H1、サブコピー、CTA、各セクション見出し |
| アートディレクター | 価格帯に見合う質感か | 写真、余白、色、タイポグラフィ |
| ウイスキー監修 | 味わい表現が自然か | 香り、味、余韻、飲み方の説明 |
| ECグロース担当 | 迷わず次の行動に進めるか | CTA、フォーム、導線、計測設計 |
| 法務・酒類販売監修 | 酒類広告として危険がないか | 年齢確認、注意書き、販売表現チェック |
| フロントエンド実装者 | 速く、崩れず、読めるか | HTML/CSS/JS、モバイル対応、PWA |

## ワークフロー

1. 目的を決める: 認知、予約、購入、問い合わせのどれを狙うか。
2. ブランド核を決める: 名前、価格帯、飲む場面、人格、禁止する表現。
3. ファーストビューを作る: 商品画像、H1、サブコピーを最初に固める。
4. セクション構成を決める: Hero, Land, Craft, Taste, Philosophy, CTA の順で流れを作る。
5. コピーを整える: 見出しは句読点なし、本文は意味の切れ目で改行する。
6. 信頼情報を足す: 味わい、製法、容量、度数、飲み方、注意表示を整理する。
7. 導線を作る: フォーム、購入ボタン、SNS、問い合わせのいずれかを明確にする。
8. Genius Review 3ループ: 第一印象、専門性、共有/購入導線の順で直す。
9. 実機確認: iPhone幅、PC幅、画像読み込み、フォーム、年齢表示を確認する。
