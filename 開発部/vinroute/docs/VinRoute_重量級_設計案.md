# VinRoute 重量級 設計書 v1.0（プロトタイプ検証済み・Claude Code引き継ぎ版）

> 目指すもの：桃鉄（移動・レース・妨害）＊モノポリー（独占の非線形報酬）＊カタン（資源配分と建設）＊ポケモン図鑑（収集の達成）を本気で掛け合わせた中〜重量級ボードゲーム。1プレイ60分級。
>
> 設計原則：**あらゆる複雑さは「悩ましい決断」に化ける限り歓迎。決断を生まない手間は足さない。**
>
> **本書の位置づけ：プレイ可能なプロトタイプ（vinroute_prototype.html）で全システムを実装・プレイテスト済み。本書の数値・ルールはすべて実装と一致しており、Claude Codeでの本格実装の正となる仕様書。**

---

## 1. コアループ（背骨）

```
サイコロを振る
  → ルートを選んで移動（道中マスでイベント／分岐が頻繁）
    → 産地に着く
      → 資源を得る／資金＋資源を払って資産を建てる
        → 畑がぶどうを生み、醸造所が量産ワイン、ワイナリーがプレミアムワイン＋試飲料を生む
          → どの産地に集中投資するか？の判断
            → 独占・地域セットで非線形ジャンプ
最終シーズン後、総価値が最大の人が勝ち
```

## 2. 資源システム（3資源）

### 2.1 資源の種類
| 資源 | 入手 | 用途 |
|---|---|---|
| ぶどう🍇 | 畑の産出・資源マス・ぶどう産地 | 醸造所・畑の建設、ワイン生産 |
| 樽🛢️ | 樽の資源マス・樽の名産地（コニャック等） | **ワイナリー専用**の希少資源 |
| 資金💰 | ルーレット式の資金マス・試飲料・都市マス | 全建設に必須＋市場で資源購入 |

### 2.2 資金はルーレット制
- 資金マスに止まると**ルーレットで金額が変動**（ギャンブル性）。
- 資金は全ての資産建設に**確実に必要**（他資源は資産により要否が違うが、資金は常に要る）。

### 2.3 樽はボトルネック資源（意図的）
- 樽はワイナリー（プレミアム生産）専用。**醸造所は樽不要**にして、ゲームが止まらないようにする。
- 樽は「上級戦略を狙う人だけが奪い合う希少資源」。詰みは市場マス（§4）で回避。

## 3. 資産システム（産地に建てる）

| 資産 | コスト | 生むもの | 役割 |
|---|---|---|---|
| 畑 | ぶどう×2 ＋ 資金 | 毎シーズン**ぶどう産出** | 資源エンジン（生産） |
| 醸造所 | ぶどう×2 ＋ 資金 | 毎シーズン**量産ワイン** | 安定した勝利点（変換）※樽不要 |
| ワイナリー | 樽×2 ＋ 資金×2 | **プレミアムワイン**＋**試飲料** | 高価値＆他人から搾取（攻めの資産） |

### 3.1 各資産の狙い（役割が全部違う＝良い複雑さ）
- **畑＝生産**：ぶどうを自給。無いと原料が尽きる必須資産。
- **醸造所＝量産**：ぶどうを量産ワイン（勝利点＆図鑑）に変換。ゲームの土台。樽が要らないので誰でも回せる。
- **ワイナリー＝プレミアム＆搾取**：
  - **プレミアムワイン**を生産（醸造所の量産ワインとは別種の高級ワイン）。
  - **試飲料**：他プレイヤーがその産地に止まると、ワイナリー所有者に資金が入る（桃鉄の物件・モノポリーの賃料）。
  - → 立地の駆け引き（人通りの多い交差点の産地にワイナリーを建てて稼ぐ）が生まれる。

### 3.2 独占（垂直統合ワイナリー）
- 1産地に畑・醸造所・ワイナリーを全部そろえると**独占ボーナス（非線形ジャンプ）**。
- 生産→量産→プレミアム＋試飲料が1か所で完結。モノポリーの独占の快感。

## 4. 市場マス（資金で資源を買える／変動レート）

- 資金を払って資源（ぶどう・樽）を買える安全弁。樽マスに止まれなくても、資金繰りでワイナリーを狙えるルートを確保（運を実力で補う）。
- **変動レート（シーズン単位・天候連動）**：
  - 毎ターンではなく**各シーズン頭に価格が決まって公開**。シーズン中は固定なので計画が立てられる。
  - 価格は**ヴィンテージ（天候）と連動**：冷夏でぶどう不作ならぶどう高騰、豊作なら暴落。樽も同様。
  - → 「今シーズンは樽が高いから樽マスを狙おう」という読みと計画が生まれる。既存のヴィンテージシステムをそのまま使うので実装が軽い。

## 5. コレクション＝ワイン（図鑑／勝利点の源泉）

- **量産ワイン（醸造所）**：その産地の標準ワイン。安定した勝利点。
- **プレミアムワイン（ワイナリー）＝ヴィンテージ銘柄**：
  - 「1985年ボルドー」のように**産地×製造年（シーズン）**で銘柄化。実在地名ベースで事実に即す（ソムリエ知識の学び）。
  - **当たり年（グレートヴィンテージ）に作ると超高価値**。天候を読んで「いつ作るか」のタイミング判断＝ギャンブル性。
- 図鑑＝集めたワイン（量産＋ヴィンテージ銘柄）の一覧。ポケモン図鑑的な収集達成。

| コレクション | 条件 | ボーナス |
|---|---|---|
| タイプコレクション | 手元に赤・白・泡・ロゼの4タイプを保有 | +50 |
| 産地コレクション | 手元のワインの産地の種類数 5/10種 | 各+8 |

| ヴィンテージコレクション | 手元の当たり年の種類 3/5種 | 各+6 |

- 設計理由：生産者は保有したいから造る。生産/保有の二重管理は不要な複雑さだった（プレイテストで判明）。
- 効果：オークションで「あの1本でコレクションが埋まる」という購入動機が生まれ、市場が回る。
- 判定タイミング：建設後・シーズン生産後・市場購入後・落札後に checkSets を呼ぶ。完成フラグ（_typeCollDone等）で二重取得を防止。

### 5.5.4 二層構造の狙い
- 地理セット（土地）とコレクション（ボトル）は対象が本当に別物なので分かれる。ボトル系は保有一本化。
- 近場で地理セットを固めるか、マップ横断で多産地のボトルを集めるか、収集目標が競合してルート選択が悩ましくなる。

## 5.8 クイズシステム（重要ポジション・確定）

**位置づけ：脇役の加速装置ではなく、ゲームの差別化の核。「遊びながらソムリエ知識が身につく」を本物にする中心システム。ただし知識ゼロでも遊べる余地を残す。**

### 選択式（4択基本）
- クイズは**選択式**（自由記述にしない）。知らなくても選べる＝初心者も参加できる。
- 基本4択。難問は6択、易問は2〜3択。
- **選択肢の数で運と知識のバランスを調整**：2択=運寄り（初心者に優しい）、選択肢が多いほど知識差が出る。

### 学習システム（間違えても学べる）
- 不正解でも**正解＋解説**が表示される。選択肢自体が学びのヒントになる。
- ゲーム中に知識が増え、終盤には全員がある程度学習して差が縮む。「遊ぶうちに詳しくなる」を体験として成立させる。

### リスク選択（自信で報酬が変わる）
- プレイヤーが難易度を選べる：2択で手堅く小報酬 / 4択・6択に挑戦して大報酬。
- 知識がある人ほど難しい方に挑んで大きく稼げる。無い人は安全策で参加できる。

### 報酬（クイズを重要にする）
- 正解で**強力な報酬**：資源の大量獲得、レアカード、注目産地の予約権 など。
- ただし**知識だけで勝たせない**：クイズを一切スルーして「移動＋資産運用」だけでも勝てる別ルートを残す（知識派と戦略派が別の勝ち方をする）。

### 両立の狙い
| 狙い | 実現方法 |
|---|---|
| クイズを重要にする | 強力な報酬源にする |
| 知識ゼロでも遊べる | 選択式＋2択の安全策＋スルーしても勝てる別ルート |
| 遊ぶうちに学べる | 不正解でも解説、選択肢がヒント |
| 教育ゲームとして唯一無二 | 「勝つために自然とワインに詳しくなる」を成立させる |

### 実装メモ
- 問題データベースは外部ファイルに分離（国データと同様、後から問題を足せる）。
- 問題の中身（具体的な設問文）は実装後に充実させる。枠組み（選択式・解説・報酬・難易度選択）を先に作る。

## 5.9 ワイナリー評価システム（パーカーポイント／確定）

**現実に即した設計：1つの産地に複数プレイヤーがワイナリーを持てる（ボルドーに多数のシャトーがあるように）。所有は早い者勝ちにせず、代わりに「評価」で競争させる。**

### コンセプト
- ワイナリー（プレミアム生産）を建てると、そのワイナリーに**評価ポイント（パーカーポイント的な100点満点評価）**が付く。
- 評価はワイン界で価格・評判を左右した実在の指標がモデル。build後も評価で価値が動き続ける＝「建てて終わり」にしない。
- 同じ産地に各自がワイナリーを持てても、評価で差がつくので競争が生まれる（流儀Bの弱点＝競争のなさを、評価で解決）。

### Q1. 評価のタイミング（定期＋たまに大物）
- **毎シーズン**：全ワイナリーが再評価される（定期・予測可能）。
- **たまに大物評論家が突然登場**：ランダムなタイミングで現れ、大きく評価を動かす（不意打ち・盛り上がり）。既存の評論家登場演出・スロットを活用。

### Q2. 評価の決まり方（運＋条件の混合）
基礎点は条件で、そこにランダムの振れ幅を加える：
- 当たり年（ヴィンテージ）に造ったワイン → 高評価寄り
- その産地を独占している（畑・醸造所・ワイナリー3つ）→ 高評価寄り
- 畑を持つ（自前のぶどう）→ 小加点
- ＋ランダムの振れ（±の運）

### Q3. 評価の効果（価値＋副次効果）
- **ワインの価値が上下**：高評価ならそのワイナリーのプレミアムワイン価値アップ、低評価ならダウン。
- **副次効果**：
  - 高評価 → 試飲料アップ（人気ワイナリーは訪問価値が高い）、注目されやすい
  - 低評価 → 試飲料ダウン

### 既存システムとの連動
- ヴィンテージ（当たり年）＝評価が上がりやすい → ヴィンテージ戦略と評価が連動。
- クイズ（ソムリエ知識）や防御カードで評価を有利にする絡め方も可能（将来）。
- 大物評論家の登場は、既存の評論家演出（スロット・登場アニメ）を再利用。

### 実装済みの計算式（確定）
- 各ワイナリーの評価 rating（60〜100にクランプ）を毎シーズン再計算：
  - 基礎80 ＋ 当たり年産地+8／不作産地-6 ＋ 同産地に自分の畑+3 ＋ 独占+5 ＋ ランダム±5 ＋ 大物評論家来訪シーズンは追加で±8
- 価値倍率：ratingToMult(r) = 1 + (r−90)×0.02（60→0.40、90→1.00、100→1.20）
- プレミアムワイン価値 = 基準25 × ヴィンテージ倍率（当たり×2／不作×0.5）× 評価倍率
- 試飲料 = 基準3 × 評価倍率。同一産地に複数ワイナリーがある場合は最高評価のみ徴収。
- 大物評論家：各シーズン約20%で来訪（🎩演出）。
- エンディングで「パーカーポイント影響」（評価等倍の基準値との差の累計 parkerDelta）を内訳表示。

## 5.10 経済システムと勝利条件（実装済み・確定）

**このゲームの competition の軸＝総資産。「コレクションだけ」だと、自分で造らずお金で名品を買うのが最適解になり、ワイナリー経営の核が空洞化する。総資産なら造る人・売る人・買う人が同じ物差しで競える。**

### 勝利条件（確定・実装済み）
- **総資産が最大のプレイヤーが勝利**。
- 総資産 = 保有ワインの価値（value）＋ 資金（money）＋ 資産価値（畑5・醸造所8・ワイナリー12）。
- コレクション等のボーナスは value に加算されるため総資産に反映される。

### ワイナリー再投資（増産）【実装済み】
- 建設済みの醸造所/ワイナリーに再投資すると生産レベルが上がる（最大Lv3）。レベルぶん毎シーズン生産。
- コスト：醸造所の増産＝ぶどう2＋資金4／ワイナリーの増産＝ぶどう1＋樽1＋資金6。

### 「売る」か「持つ」かの選択【実装済み】
- 毎シーズン末、生産した新ワインについて人間プレイヤーがモーダルで選択（NPCは量産→売る／プレミアム→持つを自動判断）。
- **卸す** → 価値の70%（SELL_RATE=0.7）で即現金。
- **持つ** → 総資産＆コレクションに満額。
- 売ると手元から消え、コレクション判定からも外れる（後述の保有ベースコレクションと連動した葛藤）。

### 都市マーケット＋オークション【実装済み】
- **都市（10都市：パリ・リヨン・ブールジュ・クレルモン・トゥールーズ・マルセイユ・ナント・ストラスブール・ディジョン他）**に止まると市場が開く。パリも通常都市（桃鉄の出発地は固定でないため特別扱いしない）。
- できること：
  - **買う**：他プレイヤーが出品したワインを購入（出品者に代金、購入者のコレクションへ、購入時にコレクション判定）。
  - **卸す**：自分のワインを70%で即現金化。
  - **競売（オークション）**：開始価格＝価値の90%。出品した瞬間、他の全プレイヤーが順に入札/パス（人間は入札UI、NPCは価値×0.8〜1.3と資金から上限を自動判断）。最高額で落札。誰も入札しなければ流れて手元に残る。
- 設計意図：90%を最低ラインに競り上がるため、「他人のコレクションに貢献するだけ」にならず出品動機が生まれる。買う側は保有コレクション完成の動機で競る。

### 実装から得た知見
- 全4段階（総資産→売る/持つ→再投資→市場/競売）の段階実装は有効だった。各段階でプレイテストして進めるのが安全。
- シャトー名（造り手）での銘柄区別は未実装（本格実装での拡張候補）。現状は「{ヴィンテージ年}年 {産地名}（評価{rating}）」で区別。

## 6. 桃鉄由来の動的イベント（マスではなくイベント連動）

### 6.1 評論家（ボンビー枠）【実装済み・確定仕様】
- **マスにしない。** 「注目産地に誰かが一番乗りした瞬間」に、**一番乗り者を除いた総資産首位**へ取り憑く（頑張った人は罰しない）。
- **毎ターンダメージ（ティア制ランダム）**：憑かれたプレイヤーのターン開始時、以下から重み付き抽選。中央に大きなポップアップ表示（人間はボタンで閉じるまでターン停止、NPCは自動表示）。
  - 軽（高頻度）：価値-2／ぶどう-1／樽-1／無害（「今日は…まあいい」）
  - 中（中頻度）：価値-5／資金-4／次のサイコロ強制1／1回休み
  - 大惨事（レア）2系統：
    - **首位狙い**：価値-15／保有ワイン全銘柄の価値2割下落／スキャンダル（資金-10＋2回休み）
    - **全員巻き込み**（持ち物が多い人ほど痛い＝相対順位にも効く）：全員のワイン価値2割下落／全員ぶどう半減／全員資金3割徴収
  - ※「全員1回休み」のような順位が変わらない一律ペナルティは意味がないため不採用（プレイテストの学び）。
- **なすりつけ**：同じマスに居合わせる or 移動中に相手のマスを通過（すれ違い）で、相手に移る。1手番1回（往復防止）。能動的な「なすりつけカード」もあり。
- 防御カード（1回無効）で防げる。
- **大物評論家**：各シーズン約20%で来訪を宣言し、そのシーズンの評価（パーカーポイント）変動が大きくなる。
- **実装上の注意**：評論家の演出モーダルはトークン管理し、古い自動クローズタイマーが新しい表示を閉じないようにする（modal競合はデッドロックの温床。プロトタイプで実際に発生し修正済み）。

### 6.2 ヴィンテージ（気候）【実装済み】
- 各シーズン開始時に当たり年産地（プレミアム価値×2）と不作産地（×0.5）を抽選・公開。次シーズンの当たり年は先読みカード用に事前計算。
- 当たり年産地＝そのシーズンの**注目産地**（スポットライト）。

### 6.3 注目産地レース【実装済み】
- 各シーズンの注目産地に最初に着いた人が+20。評論家の発動トリガーも兼ねる（首位抑制）。一番乗り者自身は評論家対象から除外。
- HUDに注目産地までの残り距離（BFS最短）を常時表示。分岐選択時は、注目産地に最も近づく選択肢に控えめな金色矢印を表示（桃鉄の目的地案内）。

## 8. マスの最終構成（実装済み）
道中マスはヴィヴィッドな単色四角（アイコンなし・色で種類を示す。凡例を画面右下に常時表示）。マスは白縁込みで道幅（7px基準）と同サイズ。産地は一回り大きい丸、都市は道中マスと同サイズの四角🏙（すべて平面はめ込み・立体ピンなし）。

- 🍇 **産地マス（丸・タイプ色）** — 資産建設＋ワイナリーワープ（後述）。色＝赤ワイン/白/泡/ロゼのタイプ。
- 🏙 **都市マス（四角・グレー）** — ワイン売買マーケット（買う／卸す／競売）。
- 🟩 **カードマス（緑 #16c65a）** — カードを引く
- 🟥 **評論家ダメージマス（赤 #ff2d2d）** — スロット演出の単発ダメージ
- 🟦 **クイズマス（青 #1f7bff）** — 選択式ソムリエクイズ（正解で報酬・解説で学べる）
- 🟪 **市場マス（紫 #b026ff）** — 資金で資源購入（変動レート）＋本日のカード
- 🟦 **資源スロットマス（水色 #00d0e0）** — スロットでぶどう1〜5/樽1〜3/資金1〜10
- 🟨 **資金ルーレットマス（黄 #ffc400）** — 重み付きルーレットで資金3〜20

### ワイナリーワープ【実装済み】
- **ワイナリー（プレミアム）を建てた産地に止まると**、自分の別のワイナリー産地へ視察ワープできる（1ターン1回・ワープ先からの連続ワープ不可）。醸造所では不可。ワイナリーの特権として差別化。

### 盤面生成の要件（実装済み・重要）
- 実在の緯度経度からフランス輪郭内にグリッドスナップ配置（桃鉄式・縦横直線）。
- 産地間は階段状ルートで接続（LINKS定義）＋メッシュで網の目化。
- **分岐は「少ない所に足す」**：基本網は控えめ（確率0.32）→一本道に分岐を足す→4方向以上の過剰分岐は連結を保って間引く。
- **全産地は最低2方向接続**（行き止まり禁止・ensureRegionConnectivity）。
- 全マス・全産地の到達可能性をビルド時に保証（プロトタイプはNodeテストで検証）。

## 9. 勝利条件
最終シーズン終了時、**総価値**が最大のプレイヤーが勝利。
```
総価値 = Σ量産ワイン価値 + Σプレミアム(ヴィンテージ)ワイン価値
       + 独占ボーナス + 地域セット完成ボーナス
       + 産地基礎価値 ± 評論家/ヴィンテージ補正
```

## 14. プロトタイプ開発で得た主要な学び（本格実装への申し送り）
1. **モーダルの待ち合わせが最重要**：非同期モーダル（売却・評論家・オークション等）は、閉じるまでターン進行を止めるコールバック設計にする。タイマーはトークン管理で古いものが新しい表示を閉じないようにする。実際に2件のデッドロックが発生し修正した。
2. **全員一律ペナルティは無意味**：順位が変わらないため。全員巻き込み系は「持ち物の量で被害が変わる」設計にする。
3. **生産と保有を分けない**：ワインコレクションは手元のボトルで判定する一本化がシンプルで、市場取引の購入動機にも直結した。
4. **盤面は密度の均等が大事**：実在の産地分布そのままだと北東に偏る。南部に産地・都市を追加し、分岐は「少ない所に足し、過剰な所は連結を保って間引く」。全産地は最低2方向接続（行き止まり禁止）。
5. **サイコロは先に確定→表示**：表示と実移動数の不一致は必ずバグ報告になる。
6. **スタック復帰の安全網**：フラグ固着時に、サイコロ再クリックで復帰できる仕組みを入れておくとプレイテストが止まらない。
7. **UI詰まりの検証はNodeスモークテストが有効**：document/canvasをモックした自動テストで、モーダルの開閉・状態遷移をコミット前に確認する。
