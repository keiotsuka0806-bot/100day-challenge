# なんで箱（Naze Box）

子どもの「なんで？」に年齢に合わせて答え、質問と回答を**家族の好奇心アルバム**として育てるWebアプリ。単なるAIチャットではなく、「あとから親が"この子が何に興味を持っているか"を振り返れる」ことを主役にした設計。

## 4つの気持ちいいMVP
0. **子ども本人が話す・聞く（音声）** — マイクで「なんで？」を話すと自動で質問。答えは自動で読み上げ＝字が読めなくても自分で使える。Web Speech API、未対応端末では自動で文字入力に戻る
1. **質問する** — 大きな入力欄に「なんで？」を書く
2. **年齢に合わせて答える** — 3/5/7/10歳で語彙・長さ・深さが変わる。回答は4ブロック（こどもへの答え／たとえ話／おうちの人へ／つぎに聞いてみよう）
3. **保存する** — なんで箱にカードでしまう（質問・年齢・カテゴリ・日付・要約）
4. **興味の傾向が見える** — 好奇心マップでカテゴリ別に集計

## 技術
- Vanilla JS（ビルド不要）/ Tailwind不使用の手書きCSS（やさしい色：白・クリーム・淡黄・淡水色）/ PWA
- AI回答は `api/ask.js`（Vercel関数 + OpenAI gpt-4o-mini）。**`OPENAI_API_KEY` が無ければモック**で動く＝公開してもコストゼロ
- 回答生成は `fetchAnswer()`（app.js）に分離＝別APIへ差し替えやすい
- 保存は `localStorage`（`loadItems/saveItems` で隠蔽＝Firebase移行しやすい構造）

## 安全設計
- センシティブな話題（死・病気・事故・性・災害など）は、子どもへの答えを最小限にし**親向け補足を厚く**する（サーバ・クライアント両方で検出）
- ユーザー入力は `textContent` / エスケープで描画＝XSSなし。鍵はサーバ関数内のみ
- レート制限（ベストエフォート）。公開時は OpenAI 側の月額上限設定を推奨

## ローカル確認
```bash
cd 開発部/naze-box
python3 -m http.server 8080   # 静的表示のみ（APIはモックにフォールバック）
# API込みで試すなら: vercel dev
```

## カテゴリ
宇宙 / 生き物 / 体 / 食べ物 / 科学 / 社会 / 感情 / その他（AI優先・キーワードでフォールバック分類）

起点＝Karpathy「文脈＝プログラム」（同じ問いでも"年齢"という文脈で出力を変える）。
