# Retrospective

今日の活動から価値ある知見を抽出し、`記憶庫/` に構造化して保存する。

## 手順

### Step 1: 今日の活動を収集する

以下を並行して読む（信頼性の高い順）：

```bash
# 1. 今日のgitコミット（最も信頼性が高い）
git log --oneline --since="$(date '+%Y-%m-%d') 00:00"

# 2. Obsidianの日報（全タブの要約が入っている）
cat "/Users/kei/Documents/Kei/$(date '+%Y-%m-%d').md"
```

git logには「何を実装したか」が、日報には「なぜそう決めたか・何を学んだか」が入っている。両方を組み合わせて分析する。

### Step 2: 3つのカテゴリに分類する

収集した情報から以下を抽出する：

**① 意思決定**（`記憶庫/decisions.md`）
- 「AかBか迷って、Aにした」という判断
- 「この方法はやめてこっちにした」という切り替え
- 将来また同じ迷いが出そうなもの

**② 学び**（`記憶庫/lessons.md`）
- 「〜だとわかった」という発見
- 「〜が原因だった」という根本原因
- 「〜すると効果的」という改善知見
- エラーの解決策と理由

**③ 再利用パターン**（`記憶庫/reusable-patterns.md`）
- 複数プロジェクトで使い回せそうなコード・構造
- 「次のプロジェクトでも使える」と思ったもの
- ボイラープレートとして定型化できるもの

### Step 3: 各ファイルに追記する

#### decisions.md への追記形式
```markdown
## YYYY-MM-DD [プロジェクト名 or トピック]
**決定**: [何を決めたか1行で]
**理由**: [なぜそうしたか]
**却下した案**: [何を却下したか・理由]
```

#### lessons.md への追記形式
```markdown
## YYYY-MM-DD
- **[Firebase / PWA / UX / CSS / 設計 など]** [学んだこと]（[なぜ重要か・いつ役立つか]）
```

#### reusable-patterns.md への追記形式
```markdown
## [パターン名]
**使い道**: [どんな時に使う]
**実装例**: `開発部/[project]/[file]` の [関数名・行数]
**注意点**: [使う際の注意]
```

### Step 4: 完了報告

保存した内容を簡潔にサマリーして報告する：

```
📚 記憶庫を更新しました

意思決定 X件 → 記憶庫/decisions.md
学び Y件 → 記憶庫/lessons.md  
パターン Z件 → 記憶庫/reusable-patterns.md

特に重要: [最も価値ある学び1つを1行で]
```

## 判断基準

**保存すべきもの**:
- 同じ問題にまた直面したとき読み返す価値があるもの
- 「なぜこうなっているか」が明確なもの
- 次のプロジェクトで時間を節約できるもの

**保存しなくていいもの**:
- コードの内容（コードはコードを読めばわかる）
- 「〇〇を実装した」という作業記録（日報にある）
- 自明なこと・ドキュメントに書いてあること

**重複チェック**:
追記前に既存エントリと重複していないか確認する。同じ学びが既に書かれていれば追記不要。
